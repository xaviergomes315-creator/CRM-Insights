Feature breakdown
Conversation list

Loads conversations 30 at a time via useInfiniteQuery (React Query v5)
IntersectionObserver on a bottom sentinel fires fetchNextPage() automatically as you scroll
Client-side search filters by contact name or phone number against already-loaded pages
Deterministic avatar colour from the conversation UUID (9 colours, consistent per contact)
Unread badge: green dot on conversations whose last_message_at is newer than the last time you opened them. Stored in localStorage per conversation ID, cleared on click
Chat window — message loading

Two-phase load: first fetches limit=1 to get total, then fetches offset = max(0, total - 50) to land on the most recent 50 messages (not the oldest)
IntersectionObserver on a top sentinel triggers loading older batches; scroll position is preserved after prepend using a scrollHeight delta calculation
Fallback "Load older messages" button for cases where the observer fires late
Date separators inserted whenever the calendar day changes between consecutive messages
Sending

Optimistic insert: message appears immediately as pending with a clock tick
On success: replaced with the server row (real id, external_id/wamid, actual status)
On failure: bubble stays with a red ✕ tick and inline error text
